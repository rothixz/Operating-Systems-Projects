Detailed Description

Problem name: Game of the rope.

(interface file)

Concept: Pedro Mariano

Synchronization based on monitors. Both threads and the monitor are implemented using the pthread library which enables the creation of a monitor of the Lampson / Redell type.

Monitor internal data structure is made visible so that the students can develop and test separately the operations carried out by the referee, the coaches and the contestants.

Definition of the operations carried out by the referee:

  • announceNewGame
  • callTrial
  • startTrial
  • assertTrialDecision
  • declareGameWinner
  • declareMatchWinner.
Author
António Rui Borges - November 2015

char assertTrialDecision ( )

Assert trial decision.

The referee computes and checks the trial result. Both the coaches and the contestants should be advised to return to their resting positions. The end of operations should determined. The internal state should not be saved.

Returns
-c 'C', if the game should continue
-c 'E', if the game is over
void callTrial ( unsigned int  t)

Call trial operation.

The referee calls the coaches to assemble the teams for a trial and waits for the teams to be ready. The trial number and the trial initial position should be updated. The internal state should be saved.

ttrial number

Declare game winner.

The referee announces which team has won the game. An error message should be generated if decision is not 'end of the game'. The game result should be updated. Both internal state and game result should be saved.
