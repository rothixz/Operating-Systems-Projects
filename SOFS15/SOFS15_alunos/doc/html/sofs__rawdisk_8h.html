Detailed Description

Access to raw disk blocks and clusters.

(interface file)

The storage device is presently a Linux file which simulates a magnetic disk. The following operations are defined:

  • open a communication channel with the storage device
  • close the communication channel previously established
  • read a block of data from the storage device
  • write a block of data to the storage device
  • read a cluster of data from the storage device
  • write a cluster of data to the storage device.

int soOpenDevice ( const char *  devname,
uint32_t *  p_bnmax 
)

Open the storage device.

A communication channel is established with the storage device. It is supposed that no communication channel was previously established. The Linux file that simulates the storage device must exist and have a size multiple of the block size.

Parameters
devnameabsolute path to the Linux file that simulates the storage device
p_bnmaxpointer to a location where the number of blocks of the device is to be stored
Returns
0 (zero), on success
-EINVAL, if devname or p_bnmax are NULL
-EBUSY, if the device is already opened
-ELIBBAD, if the supporting file size is invalid
-other specific error issued by lseek system call

Read a block of data from the storage device.

The device is organized as a linear array of data blocks. Both the physical number of the data block to be read and a pointer to a previously allocated buffer are supplied as arguments.

Parameters
nphysical number of the data block to be read from
bufpointer to the buffer where the data must be read into
Returns
0 (zero), on success
-EINVAL, if the buffer pointer is NULL or the block number is out of range
-EBADF, if the device is not already opened
-EIO, if it fails on reading
-other specific error issued by lseek system call
